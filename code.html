Understanding why transfer learning breaks down under distribution gaps (Slide 9)

1. Covariate Shift

Mathematical Form: PS(X) ≠ PT(X)

The input data distribution changes while the underlying concept stays the same. The frozen filters ($f_\theta$) encounter feature boundaries they weren't optimized to see.

Source (PS)
High-Res Natural Images
≠ Shift
Target (PT)
Low-Contrast X-Rays

2. Concept Shift (Highly Dangerous)

Mathematical Form: PS(Y|X) ≠ PT(Y|X)

The actual real-world meaning of the features has changed. Reusing a frozen backbone forces the network to map old assumptions to contradictory new labels.

X → "Car"
Source Task Context
⚠️ Conflict
X → "Anomaly"

Impact on the Network Architecture

When the Domain Divergence is broad, early layers struggle to output clean representations, forcing errors into downstream layers.

Input Data Space (x)
Early Layers (conv1 / conv2) - Generic edges & textures
Deep Extractor Layers (conv4 / conv5) - Highly specialized representations
Classification Head (gφ) - Task-specific output

The Domain Adaptation Generalization Bound

The error on your target task ($\varepsilonT$) is fundamentally restricted by the statistical divergence between your source and target domains:

εT(h) ≤ εS(h) + ½ dHΔH(DS, DT) + λ*

As the highlighted Domain Divergence (Gap) expands, the target error bound explodes, causing Negative Transfer.
